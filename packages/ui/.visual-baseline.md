# `@app/ui` — visual baseline & verification spec

Status: the `editor/components/ui` → `@app/ui` monorepo extraction is **done and
canonical** (src/ layout, `exports` map, theme in `src/styles/globals.css`,
`components.json`, ai-elements co-located). It was verified to render identically
on a cold build (no visual diff; style stays `radix-vega`).

Keep this spec for the **next** risky change — the Vega → Nova style switch —
where pixels are _expected_ to change and must be reviewed by eye (change, not
break). Re-run the coverage set below before/after that work.

## Prerequisites

- Docker running → `supabase start` (seeds users/orgs/projects via `supabase/seed.sql`).
- Editor dev server on :3000 (`pnpm --filter editor dev`); `.env.local` points at
  local Supabase + `NEXT_PUBLIC_GRIDA_USE_INSIDERS_AUTH=1`.
- Sign in at `/sign-in` (→ `/insiders/auth/basic`) as `insider@grida.co` / `password`
  (owner of org `local`, project `dev`).
- Fixed viewport (desktop preset). For a true rebuild test, `rm -rf editor/.next`
  first — a stale Tailwind cache can hide `@source`/token regressions.

## Theme & scanning (canonical wiring — what a regression would break)

- Tokens live in `packages/ui/src/styles/globals.css` (`:root` / `.dark` / `@theme`).
- The package scans its own source via `@source "../**/*.{ts,tsx}"` in that file.
- `editor/app/ui.css` does `@import "@app/ui/globals.css"` then layers editor-only
  bits (fonts, streamdown `@source`, workbench colors, animations, custom classes).
- Editor's own tree is Tailwind-auto-scanned (cwd = editor). If utility classes go
  missing app-wide, suspect this chain first.

## Seed data (deterministic, via SQL)

The dummy form is created once via dashboard **Create → Form**, then populated by SQL.
`document_id` (URL segment) ≠ `form_id` (internal; see `/form` settings). Public-form
route uses `form_id`. Connection: `psql postgresql://postgres:postgres@127.0.0.1:54322/postgres`.

```sql
-- fid = grida_forms.form.id for the dummy form
INSERT INTO grida_forms.attribute (id,created_at,updated_at,name,label,required,type,form_id,local_index,is_array,readonly)
VALUES (gen_random_uuid(),now(),now(),'name','Name',true,'text',:fid,0,false,false),
       (gen_random_uuid(),now(),now(),'email','Email',true,'email',:fid,1,false,false);
-- + 3 grida_forms.response rows (raw jsonb) and matching grida_forms.response_field rows
```

## Known capture gaps (set up before relying on a full diff)

- **`color-picker.css`** (react-colorful): only with a selected canvas node — pre-seed
  a node, then open the fill picker.
- **`sheet-without-overlay`** variant (`sidecontrol/controls/richtext.tsx`): open the
  richtext side control to trigger.
- **Database table editor** (`/data/table/[name]`): second rdg surface, editable cells.
- **Public tenant form** (`/~/{tenant}/d/e/{form_id}`): needs a published form.
- **minimal-tiptap / composer / expression-editor (@xyflow)**: custom-CSS surfaces.
