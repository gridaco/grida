// Slides (dotcanvas `editor: "slides"`) test prompts.
// The desktop home's `slides` application_preset aims these at the deck path:
// on submit, a `.canvas` slides bundle is seeded and the agent authors N slide
// documents into it, then the slides editor opens the deck. Each prompt below
// names a concrete deck subject (+ often a slide count and a style) so the agent
// can build WITHOUT a clarifying ping-pong — it should go straight to authoring
// slides, not ask "how many slides?" or "what topic?".
//
// One prompt per line. `//` lines are section headers / comments, skipped by any
// consumer. Companion structured eval set: slides-scenarios.jsonl.

// --- full deck, count + style named (strongest "just build it" nudge) ---
create a 12-slide minimalist product launch deck for a new AI note-taking app — lots of whitespace, one idea per slide.
make a 10-slide seed pitch deck for a B2B logistics startup — problem, solution, market, traction, team, ask.
build a 15-slide quarterly business review — revenue, growth, wins, risks, next quarter's plan.
draft an 8-slide conference talk on "designing for offline-first apps" — bold title slides, minimal bullets.
put together a 6-slide new-hire onboarding deck for an engineering team — culture, tools, first-week checklist.
design a 20-slide investor deck for a consumer fitness app — energetic, photo-forward, big numbers.

// --- deck with a strong visual/brand constraint ---
make a 10-slide sales deck in a dark, premium style — gold accents on near-black, serif headings.
create a 9-slide pastel, hand-drawn workshop deck about mindfulness at work.
build a 12-slide brutalist, high-contrast pitch for an underground music label.
design a 7-slide clean corporate deck in blue and white, 16:9, sans-serif, no photos — just type and simple shapes.

// --- content-from-source (turn material into a deck) ---
turn this outline into a slide deck: 1) the problem, 2) our approach, 3) how it works, 4) results, 5) what's next.
make a tidy 8-slide meeting recap deck from these notes: launch is on track; design froze the onboarding flow; engineering needs one more week for billing QA; support flagged three migration questions; marketing wants screenshots by Friday; decisions were to keep the beta invite-only and publish docs before launch; next steps are QA owner assignment, migration FAQ draft, and launch-readiness review.
summarize a 30-minute product demo into a 5-slide leave-behind for the customer.

// --- iterate / edit an existing deck ---
add a competitive-landscape slide after the market slide.
split the "how it works" slide into three steps, one per slide.
reorder so the traction slide comes right before the ask.
make every slide's headline shorter and punchier — same content.
restyle the whole deck to match a warm, editorial magazine look.
condense this 20-slide deck down to a tight 10.

// --- presentation intent ---
add speaker notes to each slide with what i should say out loud.
give me a 3-slide opening hook for a keynote — a question, a surprising stat, then the promise.

// --- ambiguous / edge (should still produce a sensible deck, sane count) ---
make me a deck about our company.
one big title slide that just says "We shipped it." — nothing else.
a really long deck — everything you know about our onboarding flow, slide by slide.
